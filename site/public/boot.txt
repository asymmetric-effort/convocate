convocate — zero-touch Ubuntu 24.04 host provisioning
=====================================================

This URL serves Ubuntu autoinstall files that take a bare machine from
ISO boot to a fully provisioned convocate host.  The result is a host
with Docker, Go, UFW, and the convocate binaries installed — ready for
either convocate-host init-shell or convocate-host init-agent.

Quick start
-----------

1. Download the Ubuntu Server 24.04 LTS ISO:
   https://ubuntu.com/download/server

2. Write it to a USB drive or mount it in your hypervisor.

3. Boot the machine from the ISO.

4. At the GRUB boot menu, press 'e' to edit the boot entry and append
   the following to the linux/linuxefi line:

     autoinstall ds=nocloud-net\;s=https://convocate.asymmetric-effort.com/autoinstall/

   Then press Ctrl+X or F10 to boot.

   Alternatively, for a PXE/iPXE boot, add the same kernel parameter
   to your PXE configuration.

5. The installer runs unattended.  It will:
   - Partition the first disk (LVM, full disk)
   - Install Ubuntu 24.04 with all convocate prerequisites
   - Create an 'operator' account (password: convocate, forced change
     on first login)
   - Install Go, clone convocate, build all binaries
   - Run convocate-host install (creates convocate user, enables Docker,
     configures UFW, sets timezone UTC)

6. After the install completes and the machine reboots, SSH in:

     ssh operator@<host-ip>

   Change the default password when prompted.

7. Depending on the host's role, run one of:

     # To set up the shell (must be done first):
     sudo convocate-host init-shell --host <this-host>

     # To set up an agent (shell must already exist):
     sudo convocate-host init-agent --host <this-host> --shell-host <shell-host>


What the autoinstall does NOT do
--------------------------------

- It does not run init-shell or init-agent.  The shell must exist before
  agents can be deployed, and the cryptographic peering requires the
  operator to specify which host is the shell.

- It does not configure static IP addresses.  Networking defaults to
  DHCP.  Reconfigure in /etc/netplan/ after install if needed.

- It does not set the hostname to anything meaningful.  The default is
  "convocate-host".  Change it with: sudo hostnamectl set-hostname <name>

- It does not inject SSH authorized_keys.  The operator account uses
  password auth initially.  Add your keys to ~/.ssh/authorized_keys
  after first login.


Files served at this URL
------------------------

/autoinstall/user-data      Ubuntu autoinstall configuration (cloud-init)
/autoinstall/meta-data      Cloud-init instance metadata
/autoinstall/late-install.sh  Post-install script (Go, convocate build)
/boot.txt                   This file


Security notes
--------------

- The default operator password is "convocate".  It is forced to expire
  on first login via chage -d 0.

- No secrets (SSH keys, API tokens, TLS certificates) are embedded in
  these files.  All cryptographic material is generated by
  convocate-host init-shell and init-agent at peering time.

- These files are served over HTTPS from GitHub Pages.  Verify the URL
  before booting: https://convocate.asymmetric-effort.com/autoinstall/
